---
title: "Pylon config reference: pylon.yaml field guide"
description: "Every pylon pipeline has its own pylon.yaml that defines its trigger, workspace, agent prompt, and notification channel. Full field reference with examples."
---

Each pylon you create gets its own config file at `~/.pylon/pylons/<name>/pylon.yaml`. Running `pylon construct <name>` creates this file through an interactive prompt; you can open it in your editor at any time with `pylon edit <name>`. Per-pylon settings take priority over the global defaults in `~/.pylon/config.yaml`, so you only need to specify fields that differ from the global configuration.

## Annotated pylon.yaml

```yaml
# Human-readable identifier for this pylon. Matches the directory name under ~/.pylon/pylons/.
name: sentry-triage

# Optional description shown in pylon list output.
description: "Sentry error triage for the nexus project"

# Set to true to skip this pylon when running `pylon start`. Useful for temporarily
# disabling a pylon without deleting its configuration.
disabled: false

# Set automatically by `pylon construct`. Do not edit.
created: 2025-06-01T09:00:00Z

trigger:
  # How this pylon is activated. Options: webhook, cron
  type: webhook
  # URL path that triggers this pylon when POSTed to.
  path: /sentry-triage
  # Optional: HMAC secret for validating webhook signatures.
  secret: "${SENTRY_WEBHOOK_SECRET}"
  # Header Pylon checks for the HMAC signature (e.g. X-Hub-Signature-256).
  signature_header: "X-Sentry-Hook-Signature"
  # Override global server.public_url just for this pylon.
  # public_url: "https://hooks.myserver.com"

  # For cron triggers, set `type: cron` and use `cron` instead of `path`:
  # type: cron
  # cron: "0 9 * * 1-5"

workspace:
  # How the agent accesses source code. Options: git-clone, git-worktree, local, none
  type: git-clone
  # Git repository URL. Use SSH for private repos.
  repo: "git@github.com:myorg/nexus.git"
  # Branch or tag to check out.
  ref: main
  # For type: local, provide the path to mount instead of repo/ref:
  # path: /home/user/projects/nexus

# Optional: override the global channel defaults for this pylon.
channel:
  type: telegram
  telegram:
    bot_token: "${TELEGRAM_BOT_TOKEN}"
    chat_id: -1001234567890
    allowed_users:
      - 987654321
  # Thread/topic subject line. Supports template variables (see prompt-templates).
  topic: "{{ .body.data.event.title }}"
  # Notification message shown above the Approve/Ignore buttons.
  message: |
    {{ .body.data.event.title }}
    {{ .body.data.event.culprit }}
    {{ .body.data.event.web_url }}
  # Whether a human must approve before the agent starts.
  approval: true

# Optional: override global agent defaults for this pylon.
agent:
  # Agent runtime. Options: claude, opencode
  type: claude
  # Auth method for this pylon's agent. Overrides the global default.
  auth: oauth
  # API key reference, used when auth is api_key.
  api_key: "${ANTHROPIC_API_KEY}"
  # LLM provider (for opencode agents).
  provider: anthropic
  # Extra environment variables injected into the agent container.
  env:
    NEXUS_ENV: production
  # The prompt template sent to the agent. Supports template variables.
  prompt: |
    Investigate this Sentry error and suggest a fix.

    Title: {{ .body.data.event.title }}
    Culprit: {{ .body.data.event.culprit }}
    Level: {{ .body.data.event.level }}
    Platform: {{ .body.data.event.platform }}
    Sentry URL: {{ .body.data.event.web_url }}
  # Maximum run time for this pylon's agent jobs. Overrides docker.default_timeout.
  timeout: "10m"
  # Host tools available to this agent. Overrides the global tools list.
  tools:
    - name: deploy
      path: /usr/local/bin/deploy.sh
      timeout: "60s"
```

## Field reference

### Top-level fields

<ParamField path="name" type="string" required>
  Identifier for this pylon. Must match the directory name under `~/.pylon/pylons/`. Set automatically by `pylon construct`.
</ParamField>

<ParamField path="description" type="string">
  Optional human-readable description shown in `pylon list` output.
</ParamField>

<ParamField path="disabled" type="boolean" default="false">
  When `true`, `pylon start` skips this pylon entirely. Use this to temporarily pause a pylon without deleting it.
</ParamField>

<ParamField path="created" type="string">
  ISO 8601 timestamp set automatically when you run `pylon construct`. Do not edit this field.
</ParamField>

### `trigger`

<ParamField path="trigger.type" type="string" required>
  How the pylon is activated. Accepted values: `webhook`, `cron`.
</ParamField>

<ParamField path="trigger.path" type="string">
  URL path for the webhook endpoint (for example, `/my-pipeline`). Required when `type` is `webhook`. Pylon registers this path on the global server port when the daemon starts.
</ParamField>

<ParamField path="trigger.cron" type="string">
  Cron expression for scheduled pylons (for example, `0 9 * * 1-5` for weekdays at 9 AM). Required when `type` is `cron`.
</ParamField>

<ParamField path="trigger.secret" type="string">
  HMAC secret used to validate incoming webhook signatures. When set, Pylon rejects requests whose signature does not match. Use `${VAR}` to reference a value from `~/.pylon/.env`.
</ParamField>

<ParamField path="trigger.signature_header" type="string">
  HTTP header Pylon reads to find the HMAC signature (for example, `X-Hub-Signature-256`). Required when `trigger.secret` is set.
</ParamField>

<ParamField path="trigger.public_url" type="string">
  Overrides `server.public_url` from the global config for this pylon only. Useful when a single Pylon server serves webhooks from multiple domains.
</ParamField>

### `workspace`

<ParamField path="workspace.type" type="string" required>
  How the agent accesses source code. Accepted values:

  - `git-clone` — Pylon clones the repository fresh for each job.
  - `git-worktree` — Pylon creates a worktree from a local clone, which is faster than a full clone.
  - `local` — Pylon mounts a local directory from the host into the container.
  - `none` — No codebase. Use this for pylons that do not need to read or modify files.
</ParamField>

<ParamField path="workspace.repo" type="string">
  Git repository URL. Use SSH (`git@github.com:org/repo.git`) for private repositories. Supports template variables so you can resolve the URL from the webhook payload (for example, `{{ .body.repository.clone_url }}`). Required when `type` is `git-clone` or `git-worktree`.
</ParamField>

<ParamField path="workspace.ref" type="string">
  Branch or tag to check out. Supports template variables (for example, `{{ .body.pull_request.head.ref }}`). Defaults to `main` when not specified.
</ParamField>

<ParamField path="workspace.path" type="string">
  Absolute path to a local directory on the host. Pylon mounts this directory into the agent container. Required when `type` is `local`.
</ParamField>

### `channel`

The `channel` block is optional. When omitted, the pylon uses the global `defaults.channel` from `config.yaml`. When present, the pylon-level settings take full priority.

<ParamField path="channel.type" type="string">
  Notification backend for this pylon. Accepted values: `telegram`, `slack`, `webhook`, `stdout`. Overrides the global default.
</ParamField>

<ParamField path="channel.telegram" type="TelegramConfig">
  Telegram connection settings. Accepts the same fields as `defaults.channel.telegram` in the global config. Required when `channel.type` is `telegram`.
</ParamField>

<ParamField path="channel.slack" type="SlackConfig">
  Slack connection settings. Accepts the same fields as `defaults.channel.slack` in the global config. Required when `channel.type` is `slack`.
</ParamField>

<ParamField path="channel.topic" type="string">
  Thread or topic subject line shown in the notification. Supports [template variables](/configuration/prompt-templates) for injecting webhook payload data.
</ParamField>

<ParamField path="channel.message" type="string">
  Notification message body displayed above the Approve/Ignore buttons. Supports [template variables](/configuration/prompt-templates).
</ParamField>

<ParamField path="channel.approval" type="boolean" default="false">
  When `true`, Pylon sends a notification with Approve and Ignore buttons and waits for a human to respond before starting the agent. When `false`, the agent starts immediately on every trigger event.
</ParamField>

### `agent`

The `agent` block is optional. When omitted, the pylon uses `defaults.agent` from the global config.

<ParamField path="agent.type" type="string">
  Agent runtime for this pylon. Accepted values: `claude`, `opencode`. Overrides the global default.
</ParamField>

<ParamField path="agent.auth" type="string">
  Authentication method. Overrides the global default for the chosen agent type. Accepted values: `oauth`, `api_key` (Claude); `none`, `api-key` (OpenCode).
</ParamField>

<ParamField path="agent.api_key" type="string">
  API key reference, used when `auth` is `api_key`. Use `${VAR}` to reference a value from `~/.pylon/.env` (for example, `${ANTHROPIC_API_KEY}`). This lets you assign a different key per pylon.
</ParamField>

<ParamField path="agent.provider" type="string">
  LLM provider for OpenCode agents. Accepted values: `anthropic`, `openai`, `google`. Overrides the global default.
</ParamField>

<ParamField path="agent.env" type="map[string]string">
  Additional environment variables injected into the agent container. Use this to pass service-specific configuration without hardcoding values in the prompt.
</ParamField>

<ParamField path="agent.prompt" type="string" required>
  The instruction sent to the agent when a job starts. Supports [Go template syntax](/configuration/prompt-templates) to inject data from the webhook payload.
</ParamField>

<ParamField path="agent.timeout" type="string">
  Maximum run time for a single job (for example, `10m`, `30m`). Overrides `docker.default_timeout` from the global config.
</ParamField>

<ParamField path="agent.tools" type="ToolConfig[]">
  Host CLI tools available to this pylon's agent. When set, this list replaces the global `tools` list entirely for this pylon. Each entry accepts `name`, `path`, and an optional `timeout`. See the [host tools guide](/configuration/host-tools) for details.
</ParamField>

## Config resolution order

When Pylon runs a job it merges settings from both files, with the per-pylon file taking full priority:

| Setting | Resolved from |
|---|---|
| `channel.type` | Per-pylon `channel.type` → global `defaults.channel.type` |
| `agent.type` | Per-pylon `agent.type` → global `defaults.agent.type` → `claude` |
| `agent.auth` | Per-pylon `agent.auth` → global agent defaults → `oauth` (Claude) |
| `agent.timeout` | Per-pylon `agent.timeout` → global `docker.default_timeout` → `15m` |
| `agent.tools` | Per-pylon `agent.tools` (if non-empty) → global `tools` |
